# 발표자료 구조 초안

## 기본 슬라이드 흐름

워크샵 참가자는 최종적으로 아래 흐름을 가진 발표자료를 만들도록 유도한다.

1. 표지
2. 아젠다
3. Copilot 기능 전체 개요
4. Step별 핵심 포인트
5. 데모 또는 before/after 비교
6. 배포 또는 운영 흐름
7. Q&A 또는 appendix

## 콘텐츠 원칙

- 한 슬라이드에는 메시지 하나를 우선한다.
- 텍스트 나열보다 구조화된 시각 표현을 우선한다.
- 키보드 이동과 발표 흐름이 자연스러워야 한다.
- GitHub Copilot 기능 소개와 결과물 변화가 연결되어야 한다.

## 구현 시점 가이드

- `Step 00`부터 `Step 03`까지는 구조만 합의한다.
- `Step 04`에서 시각 스타일과 에이전트 기반 개선을 시작한다.
- `Step 05`에서 실행, 검증, 배포 흐름을 붙인다.
- `Step 06`에서 Issue 기반 작업 위임으로 독립 기능을 추가한다.
