# UX 규칙

## 기본 규칙

- 발표자료는 데스크톱 기준으로 먼저 설계하되 모바일에서도 최소한 읽혀야 한다.
- 키보드 좌우 이동 또는 상하 이동이 명확해야 한다.
- 텍스트 대비와 타이포 위계가 분명해야 한다.
- 이미지가 없어도 메시지를 이해할 수 있어야 한다.
- 과도한 애니메이션보다 발표 흐름을 돕는 전환을 우선한다.

## 접근성 규칙

- 시맨틱 HTML을 우선한다.
- 제목 계층을 지킨다.
- 인터랙션 요소는 키보드로 접근 가능해야 한다.
- 색상만으로 상태를 구분하지 않는다.
- 이미지에는 필요한 경우 대체 텍스트를 제공한다.

## 구현 제한

- 초반 step에서는 UX 규칙만 정의하고 직접 구현하지 않는다.
- 세팅 단계에서는 규칙 파일, 프롬프트, skill 중심으로 진행한다.
