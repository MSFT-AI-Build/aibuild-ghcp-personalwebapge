# Step 03. Skills

## 1. 목표

반복 작업을 skill로 묶습니다. 여기까지도 아직 코딩 단계는 아닙니다.

## 2. 배워볼 것

- skill 구조
- 환경 점검용 skill
- QA용 skill
- 빠른 배포용 skill

## 3. Step by Step로 산출물 만들기

### Step 1. local-setup skill 다듬기

```text
`.github/skills/local-setup/SKILL.md`를 읽고,
워크샵 시작 전에 무엇을 확인해야 하는지 더 구체적으로 보완해줘.

포함:
- git 상태
- gh 로그인 여부
- 로컬 미리보기 방법
- output/ 폴더의 역할

skill 문서만 수정해줘.
```

### Step 2. slide-qa skill 다듬기

```text
`.github/skills/slide-qa/SKILL.md`를 발표자료 QA 관점에서 더 구체적으로 다듬어줘.

포함:
- 제목 구조
- 링크 경로
- 이미지 누락
- 접근성 기본 점검
- 발표 흐름 밀도 점검

output/ 구현은 하지 말고 skill만 수정해줘.
```

### Step 3. fast-deploy skill 다듬기

```text
`.github/skills/fast-deploy/SKILL.md`를 GitHub Pages 배포 관점에서 더 실전적으로 다듬어줘.

포함:
- 상대 경로
- index.html 진입점
- 배포 전 체크
- gh 활용 흐름

문서만 수정해줘.
```

## 4. 체크리스트

- [ ] `local-setup` skill이 환경 점검 흐름을 설명하는가
- [ ] `slide-qa` skill이 구조와 접근성을 다루는가
- [ ] `fast-deploy` skill이 GitHub Pages 기준을 설명하는가
- [ ] Step 03까지 `output/` 구현이 비어 있는가
